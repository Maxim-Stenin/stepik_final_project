# Вот что говорит ChatGPT:
# Какие пакеты точно нужны для твоего курса
# Для автоматизации тестов на Selenium + Pytest минимально нужны:
# pytest — чтобы запускать тесты.
# selenium — чтобы управлять браузером.
# Остальные пакеты могут понадобиться только для расширений, плагинов или специфичных задач.

# Для чего могут быть остальные
# pytest-rerunfailures — плагин для Pytest, чтобы повторно запускать упавшие тесты. Если его не используешь, можно удалить.
# attrs, exceptiongroup, iniconfig, pluggy, outcome — зависимости Pytest, их лучше оставить, иначе pytest не будет работать.
# colorama — часто используется для раскрашивания текста в консоли (Pytest иногда его использует для Windows).
# certifi, cffi, idna, urllib3, PySocks, websocket-client, h11, wsproto, trio, trio-websocket — это зависимости библиотек для сетевого взаимодействия, не обязательно для Selenium с локальным браузером, но могут понадобиться для некоторых пакетов или плагинов.
# Pygments — подсветка синтаксиса в выводе Pytest (не критично).
# packaging, sortedcontainers, typing_extensions, tomli — вспомогательные библиотеки для разных пакетов, обычно оставляют.

# Что будет, если удалить ненужные
# Если удалишь Pytest или Selenium → тесты не будут запускаться.
# Если удалишь зависимости Pytest (pluggy, attrs, iniconfig) → pytest может сломаться.
# Остальные (websocket-client, trio, wsproto и т.д.) — скорее всего, ничего критичного не произойдёт, пока ты не используешь плагины или асинхронные функции, которые зависят от них.
